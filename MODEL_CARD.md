# MODEL_CARD — Air Quality (AQI) Regressor

## 1. Назначение
Модель прогнозирует **AQI (Air Quality Index)** по концентрациям 12 загрязняющих веществ.
Цель — демонстрация полного MLOps-цикла: данные → обучение → трекинг экспериментов → деплой инференса.

## 2. Тип модели
В конфигурации по умолчанию используется **RandomForestRegressor** (sklearn).
Также предусмотрены варианты для экспериментов: `LinearRegression`, `LightGBM`.

## 3. Входные признаки
Модель ожидает 12 числовых признаков (в единицах, как в исходном датасете):

1) `PM2.5`
2) `PM10`
3) `NO`
4) `NO2`
5) `NOx`
6) `NH3`
7) `CO`
8) `SO2`
9) `O3`
10) `Benzene`
11) `Toluene`
12) `Xylene`

В API поле `PM2.5` передаётся как `PM2_5` (точка заменена на подчёркивание), далее маппится обратно.

## 4. Выход
Одно число: предсказанный **AQI**.

## 5. Метрики
Во время обучения логируются метрики регрессии:
- `mse` (Mean Squared Error)
- `mae` (Mean Absolute Error)
- `r2` (R²)

Фактические значения метрик смотри в MLflow UI (каждый run логирует метрики автоматически).

## 6. Артефакты
После обучения экспортируются артефакты для инференса:
- `models/model.joblib` — сериализованная модель
- `models/features.json` — список признаков в ожидаемом порядке
- `models/training_metadata.json` — метаданные (датасет, метрики, mlflow_run_id)

## 7. Ограничения
- Модель не использует признаки `City`, `Date`, `AQI_Bucket` (сознательное упрощение для стабильного inference).
- Качество на новых регионах/временных периодах может снижаться.
